# Purpose

This document describes real-world situations where SafeLink can help users during emergencies. These user stories guide product design and ensure that every feature solves a genuine problem.

---

# Story 1 – Student Returning Home

## User

College Student

## Situation

A student is walking home after evening classes through an area with poor mobile network coverage.

## Problem

The student feels threatened but has no reliable network to call or send messages.

## SafeLink Response

- One-tap SOS activation
- GPS location stored immediately
- Audio recording starts automatically
- Emergency alert is queued
- The app continuously searches for an available communication method
- Alert is sent automatically once connectivity is restored

---

# Story 2 – Woman in an Isolated Area

## User

Woman travelling alone

## Situation

While travelling through an isolated road, she notices suspicious activity nearby.

## Problem

There is no internet or mobile signal.

## SafeLink Response

- Emergency mode activates
- GPS location is continuously recorded
- Audio evidence is encrypted and stored
- SOS is prepared for transmission
- Future relay beacons or nearby SafeLink users may forward the alert

# Story 3 – Hiker in the Mountains

## User

Hiker

## Situation

A hiker falls and cannot return safely.

## Problem

No mobile coverage is available.

## SafeLink Response

- Offline GPS tracking continues
- Emergency evidence is stored
- SOS remains queued
- Alert is transmitted automatically when a communication path becomes available

# Story 4 – Elderly Medical Emergency

## User

Senior Citizen

## Situation

An elderly person experiences a medical emergency while walking alone.

## Problem

They may not be able to explain their condition or location.

## SafeLink Response

- Simple SOS interface
- Automatic location capture
- Emergency contacts notified
- Future wearable integration may trigger alerts automatically

# Story 7 – Industrial Worker

## User

Factory Worker

## Situation

A worker is injured inside a large industrial facility where mobile coverage is weak.

## Problem

Unable to contact supervisors quickly.

## SafeLink Response

- One-tap SOS
- Indoor location (where available)
- Future on-site relay beacons forward emergency alerts

# Story 8 – Future Smart City

## User

Citizen

## Situation

A person activates SOS in a city equipped with SafeLink infrastructure.

## SafeLink Response

- Nearest relay beacon receives the alert
- Gateway forwards it securely
- Emergency dashboard notifies responders
- Family receives live updates

# Success Criteria

Every user story should answer one question:

**Can SafeLink improve the user's chances of receiving timely assistance, regardless of network availability?**

If the answer is yes, the feature supports the mission of SafeLink.
